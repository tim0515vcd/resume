每日自動抓取匯率

類型：公司專案，獨立開發

功能：
1. 以DOS視窗為介面，抓取台灣銀行每日匯率
2. 依日期判斷一般匯率或三旬匯率，每月的5、15、25日除了要寫入一般匯率也要寫入三旬匯率
3. 今日抓取的一般匯率將寫入下一個工作日，三旬抓取的資料除了寫入下一個工作日之外，也分別將寫入11、21、隔月1日(即便是假日也寫入)
4. 搭配Windows工作排程器排定每日固定時間抓取匯率資料
5. 搭配DB中的行事曆以判斷假日或平日，假日則自動關閉程式;平日則將資料寫入DB
6. 若為週五，一般及三旬匯率將自動寫入下一個工作日，若三旬為假日則往前找上一個工作日的匯率寫入DB
簡述：

利用Python爬蟲爬取台灣銀行每日匯率，再利用Pandas擷取需要的國家匯率並整理資料，最後使用pyodbc與DB連線並寫入資料。

匯率補登

類型：公司專案，獨立開發

功能：
1. 用Python Tkinter製作介面，Menu視窗可以切換一般匯率或三旬匯率
2. 輸入欲補登日期即可按預覽按鈕，資料會顯示在下方Textbox區域，確認無誤即可在最下方匯率補登按鈕進行資料寫入
3. 若DB已有資料會跳出已有資料之通知
4. 一般匯率無輸入日期之限制，補登日期若是假日，會自動找前一工作日的匯率並寫入補登日期的下一工作日
5. 三旬匯率僅能輸入1、11或21日，程式會往前找25、5、15日的匯率並寫入DB
簡述：

為應應因為外在因素(停電、server當機...等等)導致每日自動匯率抓取程式失效或漏抓而製作，用Python Tkinter製作介面，讓User方便操做， 後端一樣是利用Python爬蟲爬取台灣銀行每日匯率，再利用Pandas擷取需要的國家匯率並整理資料，最後使用pyodbc與DB連線並寫入資料。

自動寄信小程式

類型：公司專案，與一位同事共同開發

功能：
1. 使用PyQt5設計介面
2. User登入功能
3. 需匯入信件通知表以及信件資料夾
信件通知表：格式為Excel檔，內容包含收件者(認養人)、Email、認養童案號以及是否願意使用Email寄送欄位，可從公司系統匯出
信件資料夾：內部需有兩個資料夾，分別為電子賀卡以及紙本信件，內容包含認養童給認養人的信件掃描檔並以認養童案號和認養人編號合併命名，需要User人工分類並命名
4. 按下寄信按鈕後下方TextBox會顯示信件寄出狀況，信件資料夾內，程式會自動分類資料夾內的信件， 分別將電子賀卡以及紙本信件上傳至系統並寄出Email，最後輸出Log檔到信件資料夾內。
5. User可以藉由Log檔查看是否有信件寄送失敗，進而得知成功與失敗的信件分別有幾件
簡述：

登入、檔案上傳、寄送Email功能皆利用Selenium爬蟲，爬公司內部原有的系統，代替人工完成，信件分類、編號比對、建立資料夾使用OS完成， Log檔使用txt檔輸出。

Python Python PyQt5 Python Selenium
